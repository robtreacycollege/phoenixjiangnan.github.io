Rules

Sometimes you don’t want to write rules for every single file task … so use a rule.

When you need to build a file named “main.o”

  • And “main.o” doesn’t exist, But “main.c” does …
  • Then synthesize a file task to build “main.o” from “main.c”

Rule Limitations

Rules will only do one level of inference

If lex.y exists … Rake will not build lex.c from lex.y and lex.o from lex.c.
